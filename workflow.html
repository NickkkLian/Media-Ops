二、一支视频的真实旅程（以悬案推理号 full 模式为例）

agent 之间从不对话——每一步的交接物都是文件（产物黑板）。任何一步挂掉，状态机从该步续跑，不重花钱

  1. 你👑 你：丢选题「做一支《暴风雪山庄：侦探为什么总被困在孤岛》」
  2. 脑🧠 主脑：建档派单 Claude（Opus/Fable）写入任务队列（绑定账号档案的模块组合与画风）；涉真实案件时先联网做资料包锁事实来源 → state.json research.md
  3. A1✍️ 脚本 agent DeepSeek V3 ×2-3次读 账号档案+模块指令+平台要点+资料包 → 写解说与分镜 → 内置审稿人过四关（事实/钩子/推进/画面可实现）→ 有硬伤自动修订 → script.json（含每段解说、画面描述、素材来源标记、封面底图段号）
  4. A2🎞 素材 agent DeepSeek译词 + Pexels API ‖ 与A3并行处理 A1 标 stock 的段：译英文检索词 → 素材库（新鲜度规则）→ 不够抓 Pexels → videos/NNN.mp4；找不到的段交给 A3
  5. A3🎨 生图 agent Doubao Seedream 5.0 段间 4 路并发剩余段按「画面描述+账号画风+主角参考图」生图 → images/NNN.png
  6. A4🛡 质检 agent（真自主循环） Doubao Seed 2.1 视觉评审 + Seedream 重生成读 A1 分镜逐张审 A3 的图（文字乱入/肢体畸变/与分镜不符）→ 不过关自己改写提示词重生成 → 复审，每张≤2轮 → image_qc_report.json（依赖 A3 完成，串行关口）
  7. A5🎬 动画 agent Doubao Seedance 2.0 全段一次提交·统一轮询（N段≈1段耗时）读 A4 放行的图作首帧 → 图生视频（full 模式全段）→ videos/NNN.mp4；失败自动回退动效图不卡线
  8. A6🔊 合成 agent edge-tts + ffmpeg（无LLM） 配音4路并发·编码3路并行·‖ 与A7并行汇集 A1 解说 + A2/A3/A4/A5 画面 → 配音 → 逐句字幕 → 快切/真视频铺满 → 证据卡 → BGM+响度规范 → final.mp4 subtitles.srt（实测 9 段 4 秒编完）
  9. A7🖼 封面 agent 本地渲染（无LLM）读 A1 选的强意象段底图 + 封面大字 → 渐变压暗+自动排版+强调色 → cover_final.png
  10. 脑🧠 主脑：验收打包 Claude（Opus/Fable）查各环节报告与成片抽帧 → 异常打回对应 agent 断点重跑 → 通过则打包（成片+封面+SRT+分区推荐+发布指引）并自动登记账本（模块指纹/成本/实验归属）、留痕进台账 → 发布包/ ledger.json
  11. 你👑 你：终审发布看成片 → 满意就传 B站（勾 AIGC 声明）→ published；下周报数据 → 战绩回填模块库，蜂群变得更聪明
依赖图（不是纯线性）：A1 → (A2 ‖ A3) → A4 → A5 → (A6 ‖ A7) → 打包。 质检 A4 是刻意的串行关口（画面全过审才准动画化，省 Seedance 的钱）。agent 内部也并行： 生图 4 路并发、Seedance 全段一次提交统一轮询、配音 4 路、编码 3 路（9 段实测 4 秒）。 跨任务并行（多支视频同时跑）在无人值守调度层实现（任务 #10）。
其他产线只是换零件：喜鹊周报把第 2 步换成「新闻收集+双源验证」；思想引擎换成「思想匹配（谁会想发言）」；图文线在 A1 后直接走模板渲染+违禁词质检。骨架一致。
